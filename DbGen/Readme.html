Składnia

Nagłówek pliku

1. Pierwsze linie powinne zawierać deklaracje plików wynikowych

  • Dla javy synchronicznej: 
                javaSync {
              filename JavaSync;
              package data;
              import a;
              import b;
            }
  • Dla serwera api (wymagana również deklaracja javaSync) : 
                javaApiProcessor {
              filename JavaApiProcessor;
              package data;
            }
  • Dla klienta api (obecnie dla j2j): 
                j2jApiClient {
              filename DbGenAjaxApiClient;
              package data;
              apiUrl /db/Main;
            }
    apiUrl określa ścieżkę do serwera api

2. Kolejna linia powinna zawierać scieżkę do pliku ze strukturami danych

  • Przykładowo: 
                appdata AppData.dd;

3. Kolejne linie zawierają deklaracje metod

  • Insert

    insert nazwa_metody ( [ argument : typ ]* ) "sql" ;

                insert addMessage(authorId : int, content : string)
            "INSERT INTO messages(authorId, added, content) VALUES(%[authorId], '%[content]', NOW())";
  • Update

    update nazwa_metody ( [ argument : typ ]* ) "sql" ;

                update updateMessage(id : int, content : string)
            "UPDATE messages SET contnet = [%content] WHERE  id = %[id]";
  • Query

    query typ nazwa_metody ( [ argument : typ ]* ) "sql"

    [ JOIN nazwa_pola = nazwa_metody ( [ argument : typ ]* ) [ ON parent.nazwa_pola = child.nazwa_pola] ]* ; 

                query Users getUsers(uids : int[])
            "SELECT uid, name FROM users WHERE uid = %[uid]"
            JOIN events = getEvents(this.uid) ON parent.uid = child.admin_id
            JOIN comments = getComments(this.uid) ON parent.uid = child.author_id;

Zwracanie pola typu podstawowego

        query string getName(uid : int)
        "SELECT name FROM users WHERE uid = %[uid]";

Zwracanie tablicy typu podstawowego

        query StringArray getNames(uids : int[])
        "SELECT name FROM users WHERE uid IN (%[uids])";

Zwracanie rekordu (prostego)

Codegen 
        User = record {
          uid : int,
          name : string,
        }
DbGen 
        query User getUser(uid : int)
        "SELECT uid, name FROM users WHERE uid = %[uid]";
  • Rekord zawiera tylko pola typu podstawowego.
  • W zapytaniu należy zwrócić tyle kolumn ile jest pól podstawowych w definicji rekordu

Zwracanie rekordu/rekordów (nie zawierających tablic) (możliwe również przez joiny)

Codegen 
        Image = record {
          path : string,
          alt : string
        }
        User = record {
          uid : int,
          name : string,
          image : Image
        }
        Users = array of User
DbGen 
        query User getUser(uid : int)
        "SELECT u.uid, u.name, i.path, i.alt FROM users u 
        LEFT JOIN images i ON u.image_id = i.id 
        WHERE uid = %[uid]";
        query Users getUsers(uids : int[])
        "SELECT u.uid, u.name, i.path, i.alt FROM users u 
        LEFT JOIN images i ON u.image_id = i.id 
        WHERE IN (%[uids])";
  • Rekord zawiera tylko pola typu podstawowego lub pole które jest rekordem o takich samych własnościach.
  • W zapytaniu należy zwrócić tyle kolumn ile jest łącznie pól podstawowych we wszystkich definicjach rekordów.

Zwracanie rekordów (związek 1 do 1)

Przykład trywialy

Codegen 
          Comment = record {
            tittle : string,
            content : string,
            added : string,
            author_id : int
          }
          Comments = array of Comment
          User = record {
            uid : int,
            name : string,
            comment : Comment
          }
          Users = array of User
DbGen 
          query Comments getComments(uids : int[]) 
          "SELECT tittle, content, added, author_id FROM comments WHERE author_id IN (%[uids])";
          query Users getUsers(uids : int[])
          "SELECT uid, name FROM users WHERE uid IN (%[uids])"
          JOIN comment = getComments(this.uid) ON parent.uid = child.author_id;
  • W zapytaniu należy zwrócić tyle kolumn ile jest łącznie pól podstawowych we wszystkich definicjach rekordów opórcz rekordów wypełnionych joinem.
  • this.uid odnosi się do pola zwracanego rekordu (pole uid szukane w pliku Codegena). W związku z faktem że metoda zwraca tablicę rekordów (Users), this.uid spowoduje wygenerowanie tablicy zawierającej wszystie uid-y i przekaże ją jako argument do podrzędnej metody getComments.
  • parent.uid odnosi się do pola zwracanego rekordu (pole uid szukane w pliku Codegena)
  • child.author_id odnosi się do pola pobieranego rekordu (pole author_id szukane w pliku Codegena)

API

Api udostępnia interfejs dla klienta niezależny od rodzaju połączenia z serwerem.
Obecnie generator generuje kod implementuje interfejsy:

  • Serwera (servlet java ee)
  • Klienta (j2j)

Generowanie serwera (servlet java ee)

Należy do pliku wejściowego dodać deklaracje plików wynikowych javaSync i javaApiProcessor, np: 
          javaSync {
            filename JavaSync;
            package data;
          }
          javaApiProcessor {
            filename JavaApiProcessor;
            package data;
          }
DbGen wygeneruje 3 pliki:
  • JavaSync.java
  • JavaApiProcessor.java
  • MainServelet.java
Do powyższych należy również załączyć pliki wygenerowane CodeGenem:
  • pliki struktur Api: 
                  CodeGen -json < svn/cpp/DbGen/DbGen/data/Api.dd
  • pliki AppData: 
                  CodeGen -json < AppData.dd

Generowanie klienta (j2j)

Należy do pliku wejściowego dodać deklaracje pliku wynikowego j2jApiClient, np: 
          j2jApiClient {
            filename DbGenAjaxApiClient;
            package data;
            apiUrl /db/Main;
          }
DbGen wygeneruje 3 pliki:
  • DbGenAjaxApiClient.java
  • pliki interfejsów callbacków
  • pliki klas calbacków Api (to użytkownika nie interesuje, ale musi zostać załączone)
Do powyższych należy również załączyć pliki wygenerowane CodeGenem:
  • pliki struktur Api: 
                  CodeGen -json -j2j < svn/cpp/DbGen/DbGen/data/Api.dd
  • pliki AppData: 
                  CodeGen -json -j2j < AppData.dd
